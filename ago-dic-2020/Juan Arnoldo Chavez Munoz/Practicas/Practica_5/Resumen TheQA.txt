Tipos de pruebas:
La Funcional - manual: que es donde se valida que el sistema haga lo que tiene que hacer
o también lo que no debe hacer.

Funcional - Automatizada:
es donde se utiliza un framework para automatizar las acciones manuales y poderlas hacer 
n veces de manera automática. 

No funcional - Desempeño:
se valida la capacidad del hardware, ya que se miden todos los recursos que la app
necesita para poder operar.

No funcional - seguridad:
Se utilizan herramientas para detectar las vulnerabilidades de los sistemas WEB.

Script de pruebas se define en 6 pasos básicos:
1.-Como acceder a la aplicación
2.-Como colocar las precondiciones
3.-Como ejecutar el caso
4.-Como validar el resultado esperado
5.-Como validar las post-condiciones
6.-Como salir del sistema
